## 1. 公共请求结构

所有接口共用同一外层结构（POST application/json）：

```json
{
  "request": {
    "head": {
      "funcode": "<8 字符或驼峰函数码>",
      "userid":  "U005182425",
      "reqid":   "<24 位字符串，时间戳 yyyymmddHHMMSS + 10 位随机>"
    },
    "body": { /* 各接口不同 */ }
  },
  "signature": {
    "sigdat": "<SM2 签名（dchelper 生成）>",
    "sigtim": "<yyyymmddHHMMSS>"
  }
}
```

`signature` 由 `dchelper.send_request()` 在加密后自动注入，**app.py 业务层不需要直接构造**。

## 2. 公共响应结构

```json
{
  "response": {
    "head": {
      "funcode":    "<回显>",
      "userid":     "<回显>",
      "reqid":      "<回显>",
      "rspid":      "<银行响应 id>",
      "resultcode": "SUC0000 (成功) | <其他错误码>",
      "resultmsg":  "",
      "bizcode":    ""
    },
    "body": { /* 各接口不同 */ }
  }
}
```

### 2.1 `resultcode` 关键约定（xlsx 注意事项 §1）

```
SUC0000  请求已被银行成功接收（≠ 业务成功）
其他码   需要查询接口判断业务实际状态再决定是否重发

经办类（BB1PAYOP / 代发 / 代扣等）重发规则：
  - 先调相应业务查询接口（BB1PAYQR）按 yurRef 查重
  - 确认银行无记录后才能重发
  - 重发时 yurRef 必须与原请求一致 → 否则重复提交

查询类（NTQACINF / trsQryByBreakPoint / BB1PAYQR）失败直接重发即可
```

### 2.2 已知错误码

| code | 含义 | 来源 |
|---|---|---|
| `SUC0000` | 银行成功接收 | 全接口通用 |
| `DCPG008` | 请求数据内容不正确，xxx 未提供服务（4/14 误用旧 funcode 时见到） | 4/14 失败报告 |
| `DCASY12` | 日期转换错误（测试环境 ASYCALHD 无数据时也返这个） | 4/15 报告 5.8 |

### 3.5 查询交易概要信息 `trsQryByBreakPoint`

**用途**：拉账户当日（或日期范围）实际入账/出账明细。对账主接口。

**请求 body**：
```json
{
  "TRANSQUERYBYBREAKPOINT_X1": {
    "cardNbr":   "655905978110000",
    "beginDate": "20260415",
    "endDate":   "20260415"
  }
}
```

**响应 body**：

```json
{
  "TRANSQUERYBYBREAKPOINT_Y1": [
    { "acctNbr": "...", "expectNextSequence": "5", "transDate": "20260415" }
  ],
  "TRANSQUERYBYBREAKPOINT_Z1": [
    {
      "queryAcctNbr": "655905978180000",
      "ctnFlag":      "N",
      "creditAmount": "0.00",  "creditNums": "0",
      "debitAmount":  "-0.04", "debitNums":  "4"
    }
  ],
  "TRANSQUERYBYBREAKPOINT_Z2": [
    {
      "transDate":         "20260415",
      "transTime":         "102100",
      "transSequenceIdn":  "C0547EP000056BZ",  // 交易流水（DCSIGREC 入参）
      "loanCode":          "D",                // D=借/出 C=贷/入
      "transAmount":       "-0.01",
      "ctpAcctName":       "殷萌涵",
      "ctpAcctNbr":        "6214837811201866",
      "remarkTextClt":     "供应商货款测试-滇界云管",
      "yurRef":            "DJ-TEST-20260415-4131095"
    }
  ]
}
```

| 字段 | 含义 | 备注 |
|---|---|---|
| `expectNextSequence` | 下次续传序号 | 大数据分页用 |
| `ctnFlag` | 是否还有下一页 | `N` = 无后续 / `Y` = 需要续传 |
| `creditAmount/Nums` | 总贷方金额/笔数 | 入账汇总 |
| `debitAmount/Nums` | 总借方金额/笔数 | 出账汇总 |
| `transSequenceIdn` | **交易流水 idn** | DCSIGREC `trsseq` 入参 |
| `loanCode` | 借贷标志 | `D` 借（出）/ `C` 贷（入） |
| `yurRef` | 业务参考号 | 自动付款的可按此匹配 PaymentSchedule |

## 6. yurRef 防重设计（应对 WARN-2）

**协议要求**：经办类请求重发必须用相同 `yurRef`，否则重复提交风险。

**V4 实现**（cmbPayment.ts + paymentSchedule.ts 串通）：

```ts
// scheduleId 全局唯一 = 天然 yurRef
const bizNo = String(scheduleId)

// 第一次发: BB1PAYOP yurRef=scheduleId
const resp1 = await cmbTransfer({ ..., bizNo })

// 网络超时 / 未确认 → 同 yurRef 调 BB1PAYQR 查
const queryResp = await cmbQueryPayment(bizNo)
if (queryResp.found && queryResp.reqSts === 'BNK') {
  // 银行已收 → 不重发，按已成功处理
} else {
  // 银行无记录 → 同 yurRef 重发 BB1PAYOP
  await cmbTransfer({ ..., bizNo })   // 同 bizNo，绝不变
}
```

**绝对禁止**：
- 重试时改 `yurRef`（即使加后缀也不行）
- 同一笔 schedule 用多个 yurRef
- 跨 schedule 复用同一 yurRef
